# Core Overview – Was dieser Bereich regelt

Der `00-core`-Bereich definiert den „Betriebssystem-Kern“ des Project Manu Core.

Er legt fest:
- wie dieses Repo gelesen werden soll,
- wie neue Inhalte eingebunden werden,
- welche Konventionen für Dateien und Ordner gelten,
- wie KIs sinnvoll mit dem Repository arbeiten,
- wie Informationen wachsen, ohne chaotisch zu werden.

## 1. Ziel

Ein Repository, das:

- **langfristig erweiterbar** ist,
- **für verschiedene KIs** nutzbar ist (ChatGPT, Gemini, Claude, …),
- **für Menschen** lesbar und nachvollziehbar bleibt,
- **ohne doppelten Müll** auskommt,
- und in dem Inhalte **auffindbar**, **kontextklar** und **wiederverwendbar** sind.

## 2. Zentraler Gedanke

Dieses Repo ist keine statische Ablage, sondern ein **lebendiges Arbeitsgedächtnis**.

Das heißt:
- Inhalte dürfen wachsen.
- Strukturen dürfen nachjustiert werden.
- Aber: Die Grundlogik (Core → Persona → Projekte → Wissen → Assets) bleibt stabil.

## 3. Audience

Der Core richtet sich an:
- Manu selbst,
- zukünftige Versionen von KI-Systemen,
- evtl. menschliche Mitleser:innen, die mit diesem Repo arbeiten.

Alle Texte in `00-core` sind so geschrieben, dass sowohl Menschen als auch KIs damit arbeiten können.
